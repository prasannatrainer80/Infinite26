1) Collections : Arrays are static data structures, means once assigned we cannot alter the size. But, if you want to add the elements irrespective of size, then we need the help of collections. 

Collections can grow to any size unlike arrays. 

Collections are from java.util package.

Collections stores data in Object format. 

The top 3 interfaces of collection framework are 

(i) List : Allows you to store list of values. 

ArrayList, Vector, LinkedList are falling under List interface

(ii) Set : Allows you to eliminate duplicate entries from the given collection. HashSet and LinkedHashSet are falling under set Interface

HashSet displays records after removing duplicates in Hashing order, where LinkedHashSet used to display records in Insertion Order. 

(iii) SortedSet : Allows you to display records in Sorted Order. TreeSet is falling under SortedSet interface.

Map Interface : Allows you to store data w.r.t. Key/Value Pairs. Here, the key cannot be duplicated or null, but value can be repeated. 

HashTable and HashMap are falling under Map interface. 

Generics : Provides an opportunity to protect data. If you want to define data type at runtime then we need the help of generics. 

Generics will be represented using <T>, as in place <T>, we can pass any value type/ref type or custom type. 

Generic methods ensures write once and apply any data type. 

Advantages :
___________


1) Write once and apply for all types

2) Provides security to the data

3) No Typecasting required

4) Reduces the size of the program.
